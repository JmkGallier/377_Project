The proposed system solves this informational problem because it gives the stakeholders a platform where they can access Prince George’s County spending data for the year 2018. We believe this will have a big impact of the shareholders because the shareholders are tax payers and are interested to see where their money is being spent. This will also help shareholders conduct research to help optimize the spending budget of Prince George’s County. Explain which API you've chosen to work with, and why (what's your information problem)? We chose to work with the “Spending Information for FY 2018” API. The reason why we chose to work with this API is because many ordinary shareholders will not access this data because they don’t know how to. By creating a simple platform for them, they will be better educated and help optimize the Prince George’s County budget. PG County Finance/Budget Spending Information FY2018 API Link: https://data.princegeorgescountymd.gov/Finance-and-Budget/Spending-Information-for-FY-2018/2qma-7ez9/data A description of the front-end technology you expect to use to make your application Our web application will use HTML, CSS, and JavaScript to structure the webpage and visualize the data. JavaScript will also be used Presentation Proposal We will be approaching dataset visualization in two ways by visualizing county-wide expenditure per agency and agency expenditure per zip code. Expenditure per agency will be done by using a Voronoi Treemap which can be thought of as a pie chart with minor segments within each major segment. Each major segment will illustrate an agency by relative expenditure amount with the description of the payment making up the subdivisions of each agency, illustrating expenditure by size.
